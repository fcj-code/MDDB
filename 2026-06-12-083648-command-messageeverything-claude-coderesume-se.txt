  根因

  // 修改前：Obsidian 切 tab 回来时再次调用处理器，el 已有内容
  this.registerMarkdownCodeBlockProcessor('mddb-table', (source, el) => {
    // 直接 el.createEl(...) — 不会清空 el
    // 所以第二次调用时，旧内容和第一次调用的结果都在
    // → 双重解析：两个表格、两套下拉选项
  });

  Obsidian 的行为机制：
  - registerMarkdownCodeBlockProcessor 注册的处理器在 leaf 
  激活/切换时会被重新调用
  - 但传入的 el 元素是 同一个 DOM 节点（Obsidian 复用容器）
  - 没有 el.empty() 时，新内容追加到旧内容后面 → 双份渲染

  这是 Obsidian
  插件开发的常见陷阱。官方推荐的最佳实践就是在处理器入口处始终先清空容器。

  修复

  在 src/main.ts 的两个处理器入口各加了一行：

  ┌────────────┬──────┬────────────┐
